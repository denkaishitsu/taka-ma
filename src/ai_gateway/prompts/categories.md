判定は独立した 2 軸で行う（旧 light/heavy の 1 次元 2 値は廃止）。

### execution 軸（実行方式 — 写像テーブル routing.matrix の入力軸。レーンは写像後モデルの実行 method が決める）

- **inline**（純生成・単発。ローカル軽量モデルが 1 回のプロンプト応答で完結させる）
  - 単純な質問応答
  - テンプレート・ボイラープレート・設定ファイルの生成
  - 定型的な文面・スニペットの純生成
  - ※ ファイル読み取り・フォーマット変換は inline に含めない（ツール文脈が要るため agent へ）
- **agent**（対話型エージェント。探索・試行錯誤・ツール使用・対話反復を伴う）
  - ファイルを読み、実行し、エラーを見て修正する作業
  - ツール使用（ファイル検索、コマンド実行、テスト実行）を伴う作業
  - コード生成・バグ修正・単体テスト作成のうち、読み書き/検証を伴うもの
  - 要件定義・設計・実装・コードベース解析・アーキテクチャ判断
  - 判断に迷うもの全て

### depth 軸（深さ — agent の場合のみ意味を持つ。写像テーブルでモデル階梯を決める）

- **shallow**（浅い・定型的なエージェント作業。1〜数手で終わる）
- **deep**（深い設計・難所。探索と試行錯誤が本質的なもの）
- **省略**（判断がつかない場合は depth を出さない → 中位モデルに落とす）

inline の場合、depth は不問（省略してよい）。
